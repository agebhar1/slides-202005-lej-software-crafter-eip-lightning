Messaging

  • async, program-to-program communication
    • reliable delivery
  • sending packets of data
    • messages w/ header & body
  • channels (a.k.a. queues) connect programs
    • convey messages
    • possible shared across multiple computers
  • sender (/producer)
  • receiver (/consumer)

Messaging System

Why?

  • more immediate than File Transfer
  • better encapsulation than Shared Database
  • more reliable than Remote Procedure Invocation
